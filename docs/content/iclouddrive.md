# {{< icon "fa fa-cloud" >}} iCloud Drive

## Configuration

Modern Apple login requires an anisette provider.

One straightforward way to run it is with Docker:

```console
docker run -d --restart always --name anisette-v3 -p 6969:6969 --volume anisette-v3_data:/home/Alcoholic/.config/anisette-v3/lib/ dadoum/anisette-v3-server
```

This can be run either as your normal user, if your Docker setup allows that,
or with `sudo` on systems where Docker requires root privileges.

By default the backend uses `http://localhost:6969` as the anisette URL. If
your anisette server is running somewhere else, set `anisette_url` in advanced
config.

The initial setup for an iCloud Drive backend involves getting a trust token/session.
This can be done by simply using the regular iCloud password, and accepting the code
prompt on another iCloud connected device.

**IMPORTANT**: At the moment an app specific password won't be accepted. Only
use your regular password and 2FA.

`rclone config` walks you through the token creation. The trust token is valid
for 30 days. After which you will have to reauthenticate with `rclone reconnect`
or `rclone config`.

After you approve the 2FA prompt, rclone may take a few extra seconds to finish
while Apple stages the iCloud Drive PCS cookies required for ADP-enabled
accounts.

Here is an example of how to make a remote called `iclouddrive`.  First run:

```console
rclone config
```

## Advanced Data Protection

ADP is supported.

When ADP is enabled, rclone may need approval from one of your trusted Apple
devices before iCloud Drive can be accessed on the web. Keep `Access iCloud
Data on the Web` enabled in Apple Account settings, then approve the request on
your trusted device when prompted.

The first iCloud Drive command after login may pause briefly while Apple stages
the web cookies needed for PCS access. Once that completes, later commands
should reuse the saved session.

## Troubleshooting

### Waiting for trusted device approval

If ADP is enabled, rclone may pause while it waits for approval on a trusted
Apple device. Approve the web access request, then retry if the command timed
out before the approval completed.

If you recently changed `Access iCloud Data on the Web` or the ADP setting,
clear the saved `cookies` and `trust_token` values or run
`rclone reconnect remote:` to refresh the session.
